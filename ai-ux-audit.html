UX + AI Concept

AI UX Audit

A personal project exploring how AI can deliver instant, actionable UX feedback to developers and designers who need a second opinion — fast, free, and without needing to ask anyone.

TypePersonal Project
RoleUX Designer + AI Workflow
ToolsFigma, Claude AI
StatusConcept / MVP
01 — Problem

Most web interfaces ship without any UX review

A lot of developers and early-stage teams build and launch products without any formal UX feedback. They don't have a designer on the team, don't know how to evaluate heuristics, and don't want to bother a senior designer or wait days for a response.

Junior designers face the same problem — they need validation on their work but feel uncomfortable asking their manager for every small decision.

The core problem

  • Developers build UIs without knowing basic UX heuristics
  • Junior designers need fast feedback without escalating every decision
  • Startup founders have no budget for a UX consultant
  • No fast, free, zero-friction way to get actionable UX feedback exists
02 — Context & Idea

Using AI to bring a design tool idea to life

This project started as a personal experiment. I wanted to explore how far I could take an idea using AI-assisted workflows — from concept to a working design. The goal wasn't just to build something, but to understand how AI can accelerate the design process without losing craft.

The concept: a tool where you upload a screenshot of your UI, and the AI analyzes it against UX heuristics — giving you a score and a prioritized list of fixes ranked by impact and effort. No account needed. Results in under 15 seconds.

03 — Users

Who needs this

Developers without design knowledge

  • Build UIs but have no UX training
  • Want to improve their interfaces but don't know where to start
  • Need feedback that's concrete, not vague

Junior designers & startup founders

  • Need validation without asking the team every time
  • Have limited time and resources
  • Want to prioritize what to fix first
04 — User Flow

As simple as possible

The main design decision was to keep the flow to the absolute minimum. No sign up, no configuration, no learning curve. One action leads to one result.

06 — Design Decisions

Key decisions & what was discarded

Web app — no account required — lowest friction, maximum reach

  • Results prioritized by impact & effort — not just a list of issues
  • Score out of 100 — gives users a clear, comparable metric
  • No account required — removes the biggest drop-off point
    • Single action to result — one upload, one output

    What was discarded

    • Chrome extension — adds installation friction, limits reach
    • Real-time feedback as you design — too complex for an MVP
    • User accounts and history — unnecessary for the core value
    • Generic AI feedback without heuristic structure — not actionable enough
    07 — Final Design

    From wireframe to live product

    The final design kept the same principles as the wireframe — clean, minimal, focused on the task. The visual layer added brand, hierarchy, and the right amount of color to communicate severity and priority without overwhelming the content.

    Landing — AI-powered UX review tag, headline, sample report, stats

    Audit form — upload, screen type, user goal, UI text

    Loading state — 6-step checklist with progress bar

    Result screen — UX score, key insight, top fixes ranked by ROI

    08 — Error States

    Handling failure gracefully

    Since the tool is form-based, error handling was a key part of the UX. Four error states were defined in MVP scope:

    Invalid input

    • User submits without a screenshot. Clear inline message pointing to the required field.

    Something went wrong

    • AI analysis fails or times out. Clear message with option to retry, not a blank screen.

    Unsupported file format

    • User uploads a PDF or video. Friendly message specifying accepted formats.

    Image too small

    • Screenshot resolution too low. User prompted to upload a higher quality image.
